# 🕋 منصة تحفيظ القرآن الكريم (Tahfez Quran Platform)

منصة رقمية إسلامية متكاملة (Full-Stack) وعصرية لإدارة حلقات تحفيظ وتجويد القرآن الكريم. تتميز المنصة بتصميم عربي ملكي بلمسات الزمرد والذهب، واتجاه واجهات كامل من اليمين إلى اليسار (RTL)، مع صلاحيات أدوار تفصيلية (RBAC) لـ (الطالب، ولي الأمر، المعلم، المدير).

---

## ✨ المميزات الرئيسية للمنصة

1. **الواجهة التسويقية الفاخرة**: تصميم فريد بألوان دافئة، عرض الباقات والمميزات، نموذج تواصل وتجربة استخدام متجاوبة بالكامل.
2. **نظام المصادقة والتحقق (OTP)**: تسجيل مستخدمين بصلاحيات مختلفة وتفعيل الحسابات وتغيير كلمة المرور عبر كود التحقق.
3. **لوحة تحكم الطالب**: تتبع تقدم الحفظ والمراجعة بالآيات والأجزاء مع رسم بياني للتسميع اليومي.
4. **لوحة تحكم ولي الأمر**: متابعة الأبناء وتلقي إشعارات بتقدمهم وعلاماتهم وإدارة الاشتراكات والدفع.
5. **لوحة تحكم المعلم**: إدارة الحلقات، جدولة الحصص، رصد غياب الطلاب، وتقييم الحفظ والتجويد.
6. **الحصص المباشرة (Jitsi Integration)**: فصول تفاعلية مباشرة (صوت، صورة، مشاركة شاشة) مدمجة بالكامل داخل المنصة.
7. **الشهادات الفاخرة للطباعة**: توليد تلقائي لشهادات تفوق الطلاب بنقاط تسميع ممتازة بصيغة قابلة للطباعة المباشرة.
8. **نظام الفوترة والاشتراكات**: سداد باقات الحفظ (شهري/ربع سنوي) مع تتبع الفواتير التاريخية.
9. **الوضع الداكن**: إمكانية التبديل بسلاسة بين الوضعين الفاتح والداكن للحفاظ على راحة العين أثناء القراءة والحفظ.

---

## 🛠️ التقنيات المستخدمة (Tech Stack)

* **الواجهة الأمامية (Frontend)**: Next.js 14+ (App Router), TypeScript, Tailwind CSS v4, Cairo & Amiri Google Fonts.
* **الواجهة الخلفية (Backend)**: RESTful API مبني بـ Node.js + Express + TypeScript, Prisma ORM, Swagger OpenAPI.
* **قاعدة البيانات**: PostgreSQL.
* **إدارة الحاويات**: Docker / Docker Compose.

## 🚀 كيفية التشغيل والتشغيل التجريبي

تأكد أولاً من تثبيت [Node.js](https://nodejs.org) و [Docker](https://www.docker.com) (في حال الرغبة بالتشغيل بالحاويات).

### الخيار الأول: التشغيل الفوري باستخدام Docker Compose (موصى به)

بأمر واحد فقط، يمكنك بناء وتشغيل كامل خدمات النظام (الفرونت إند، الباك إند، قاعدة البيانات، خادم Redis):

```bash
docker-compose up --build
```

* واجهة المستخدم (Next.js) ستكون متاحة على: `http://localhost:3000`
* واجهة البرمجة (REST API) ستكون متاحة على: `http://localhost:5000`
* توثيق Swagger للـ API سيكون متاحاً على: `http://localhost:5000/api/docs`

---

### الخيار الثاني: التشغيل المحلي بدون Docker (التطوير المحلي)

#### 1. تشغيل قاعدة البيانات والباك إند:
* ادخل لمجلد `backend`:
  ```bash
  cd backend
  ```
* ثبّت الحزم المطلوبة:
  ```bash
  npm install
  ```
* انسخ ملف الإعدادات وقم بتعديل رابط الاتصال بقاعدة البيانات الخاصة بك في ملف `.env` (يدعم PostgreSQL أو SQLite):
  ```bash
  cp .env.example .env
  ```
* قم بتوليد عميل Prisma وبناء المشروع:
  ```bash
  npx prisma generate
  npm run build
  ```
* شغل خادم الباك إند في وضع التطوير:
  ```bash
  npm run dev
  ```

#### 2. تشغيل الفرونت إند (تطبيق Next.js):
* افتح نافذة كونسول جديدة وادخل لمجلد `frontend`:
  ```bash
  cd frontend
  ```
* ثبّت الحزم المطلوبة:
  ```bash
  npm install
  ```
* شغل خادم التطوير للواجهة:
  ```bash
  npm run dev
  ```
* افتح المتصفح على الرابط `http://localhost:3000` للدخول للموقع وتجربة كامل خصائصه.
